---
name: grilling
description: Interview the user relentlessly about a plan, decision, or idea. Use when intent is vague, underspecified, or the user wants to stress-test their thinking.
---

One question at a time. Walk the decision tree branch by branch, resolving dependencies in order. Offer a recommended answer with each question. If a fact is code-answerable, look it up instead of asking. If a fact depends on external information (library behavior, API specs, current events), search the web instead of asking.

Do not act until the user confirms shared understanding.
